## 2 · DECISION LOG (append-only)

| ID | Date | Decision | Reasoning / Alternatives / Risks |
|---|---|---|---|
| DEC-021 | 2026-07-23 | Ideation is **demand-governed with windowed idempotency** (`ideate:{acct}:{bucket}`, bucket = now // IDEATE_COOLDOWN_S) rather than slowing the tick | Tick also drives sweeps/carousels/CFO checks — slowing it breaks other duties; windowed keys make floods *structurally* impossible independent of cadence. Alternatives: slower tick (rejected), DB unique constraint (needs migration). Risk: window too long delays refill after clears → mitigated: governor re-checks need every tick; the window only caps *job creation*. |
| DEC-022 | 2026-07-23 | SLA classification uses a **simple minutes-per-post heuristic** (45 min default, env-tunable), not learned cycle times | Zero data exists on real cycle times (0 published); heuristic is honest + tunable; learning version deferred until real throughput data accumulates. Alt: percentile ETA from job history (no history yet). Risk: mis-tuned estimate only affects at_risk earliness, never breach correctness. |
| DEC-023 | 2026-07-23 | Fairness implemented as **candidate-page reorder**, CAS untouched | Preserves proven multi-worker safety; O(page) cost; degrades to old order on any error. Alt: SQL-side ordering (supabase-py can't express it); per-account queue tables (Phase 7). Risk: page-local fairness only — acceptable at current volumes; Phase 7 supersedes. |
| DEC-024 | 2026-07-23 | Paused prep = **free_chat-only, hard-guarded, test-enforced** | Founder's $0 mandate for paused accounts; a source-level test asserts no paid path is reachable from `_run_task`. Alt: cheap-tier routing (could leak paid calls). Risk: free providers all down → prep silently skips (correct: $0 > progress). |
| DEC-025 | 2026-07-23 | Prep scripts stored as board `status='prep'`, excluded from quota/in-flight | Keeps prepared inventory visible in Studio without distorting live production math. Alt: separate table (migration + UI work). Risk: Studio needs a 'prep' filter chip (REQ-SLA-UI). |
| DEC-026 | 2026-07-23 | The Master Ledger lives **in the repo** (docs/LEDGER.md), updated in every release zip | The chat-only ledger was lost when the prior session was cut off mid-delivery and the sandbox reset — a repo file survives both. Alt: DB table (harder for founder to read/diff). Risk: none material; requires each zip to include the updated file. |
