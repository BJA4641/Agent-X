 WHAT'S NEW IN v5.4
 ------------------
 1. FIXED: pet posts no longer get #ai #tech hashtags
     - brain.py demo script was hardcoded AI content → now niche-neutral
     - _normalize_script() now accepts account_id and looks up niche
     - Hashtag fallbacks are niche-aware (pets, cats, dogs, fitness, finance,
       cooking, ai, tech, beauty, gaming, travel dictionaries)
     - Visual prompts no longer default to "dark tech-noir" for non-tech posts
     - Caption/sound defaults are niche-appropriate
 2. FIXED: scout trends are now filtered by niche when generating a script;
    a cat account gets cat trends, finance account gets finance trends.
 3. FIXED: Stop-work auto-reject repair in SQL — 84 rejected items that
    were auto-rejected (no human reason) get restored to 'idea' pool.
 4. ADDED: 74 AI models across 7 categories — unified provider catalog:
     • Text/chat: 19 models (Claude, GPT-4o/o3, Gemini, DeepSeek, Grok,
       Cohere, Mistral, Groq, Llama, Qwen, Kimi K2, Fireworks, Together)

       Pollinations FREE, Hunyuan)
     • Image→Image (edit): 6 models (GPT edit, FLUX Kontext, FLUX Fill, SD inpaint,
       Ideogram Remix, Gemini edit FREE)

       Minimax Hailuo, Luma Dream Machine, Vidu Q1, Hunyuan, LTX, CogVideoX)

    so failed keys automatically rotate to working ones.

    status and chosen defaults.
 7. ADDED: agentcore/aisuite.py — new unified router that every new worker

 ==========================================================
 STEP 1 — SUPABASE (REQUIRED)
 ==========================================================
 In Supabase SQL Editor, open and run:
    db/v5.4_PRODUCTION.sql
 (It's the fixed v5.3 SQL with v5.4 additions; fully idempotent.)

 Then verify:
    SELECT value FROM settings WHERE key='schema_version';
 Expected:  {"v": "5.4", "applied_at_epoch": ...}

 This will:
   - Create any missing v5 tables (jobs, agent_events, memory, etc.)
   - Add provider_label + cost_cents columns to run_ledger
   - Add model selection rows (model_t2i, model_t2v, etc.)
   - RESET auto-rejected board items back to 'idea' pool (the 84 rejections
     that happened when you paused the pet project)

 ==========================================================
 STEP 2 — DEPLOY CODE
 ==========================================================
 A. Push the web/ + pipeline/ + db/ folders to GitHub.
 B. Vercel auto-deploys web.
 C. Railway auto-deploys worker (Dockerfile CMD = python cli.py worker).
 D. In Railway, add any new API keys you want from the list above.
    Start with FAL_KEY if you want video models (~$1 free credit on signup).
    GEMINI_API_KEY alone unlocks free text + free images + free TTS.

 ==========================================================
 STEP 3 — VERIFY
 ==========================================================
 1. Open /dashboard/models — you should see 7 category tabs, each with a
    grid of models. The first time most will say "NO KEY" except Gemini
    (FREE) and anything you already have keys for.
 2. Open /dashboard/console — "AI provider wallets" shows live balances
    for the keys you added; "Auto-fallback ON" banner is visible.
 3. Unpause the pet account; resume worker. New scripts will have pet
    hashtags (#catsoftiktok, #petrescue, etc.) NOT #ai.
 4. Check agent activity feed — scout messages should say "pets: N hot items"
    instead of generic AI/finance topics.

 TROUBLESHOOTING
 - "relation does not exist": re-run db/v5.4_PRODUCTION.sql (idempotent).
 - Models show NO KEY: add the env var in Railway, redeploy.
 - Videos aren't generating: add FAL_KEY (1 key unlocks Veo3, Kling, Runway,
   Pika, Hailuo, Luma, Vidu, LTX all at once).
 - Legacy loop firing: confirm Railway CMD is `python cli.py worker` (not `loop`).
