================================================================
WHAT'S NEW IN v5.1 (Phase 3 — production optimization)
================================================================

1. AUTO-THROTTLE (fixes "burn entire budget by noon")
   - When >90% of daily budget is spent, low-priority jobs are
     delayed 30-120 minutes instead of being killed outright.
   - Keeps posts trickling out through the whole day instead of a
     burst-and-silence pattern.
   - ON by default. Toggle via settings.autothrottle row.

2. WORKER HEARTBEATS + CEO DASHBOARD
   - Worker writes a heartbeat to worker_health every 30s with
     completed/failed/in-progress counts.
   - New /dashboard/ceo page shows everything live (refreshes 15s).
   - You can see at a glance if the worker is alive, how many jobs
     are in each state, and how budget is tracking.

3. HUMAN DESK (escalations queue)
   - When an agent raises HumanEscalation (sponsorships, ban-risk,
     big decisions), a row lands in escalations table.
   - /api/human lists open escalations; POST resolves them and
     the originating job is re-queued.
   - Founder approval UI wired up end-to-end (future v5.1.x adds
     the inline resolution widget to the workspace page).

4. EXPERIMENTS ENGINE (self-improvement start)
   - experiments table tracks A/B hook variants.
   - experiments.create_hook_test creates a 24h experiment.
   - experiments.decide picks a winner after 24h and records a
     typed lesson (confidence-weighted) into the lessons table.
   - Later in Phase 3.x this feeds the prompt-evolution loop.

5. KPI SNAPSHOTS
   - ops.snapshot writes hourly spend/publish/views/inflight rows
     to kpi_snapshots so we can build trend charts in v5.2.

6. RAILWAY DEPLOY RESILIENCE
   - Every "self-scheduling" job uses idempotency keys so Railway
     restarts / redeploys / cold starts don't create duplicate
     heartbeats/snapshots/syncs.
   - Worker tracks completed/failed counters and reports them in
     the heartbeat.
